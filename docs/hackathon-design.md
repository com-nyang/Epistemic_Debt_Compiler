# Epistemic Debt Compiler — 해커톤 설계

> **핵심 메시지:** "We don't stop agents from acting. We stop them from acting without evidence."

---

## 1. 제품 한 줄 설명

AI 코딩 에이전트가 근거 없이 고위험 행동(파일 수정, 명령 실행)을 할 때 이를 가로채고, 증거가 생길 때까지 실행을 지연시키는 CLI 게이트키퍼.

---

## 2. 핵심 문제 정의

AI 에이전트는 "이렇게 하면 될 것 같습니다", "아마 이 부분이 원인일 것입니다" 같은 **추정을 사실처럼** 다음 행동의 전제로 쓴다. 이 추정이 쌓이면 에이전트는 근거 없는 수정 → 근거 없는 테스트 패스 주장 → 근거 없는 배포 제안으로 이어지는 **인지부채 연쇄**를 만든다.

문제는 에이전트가 "틀렸다"는 게 아니다. **근거 없이 행동했다**는 것이다.

---

## 3. 핵심 사용자 시나리오

### 시나리오 A — 단독 에이전트 사용자

```
사용자가 Claude Code에게 버그 수정을 요청.
에이전트: "이 함수가 원인인 것 같습니다. 수정하겠습니다."
→ EDC가 "같습니다" 패턴을 인지부채로 기록
→ 실제 파일 Edit 직전 인터셉트: "인지부채 2건. 테스트 실행 후 진행하시겠습니까?"
→ 사용자가 테스트 실행 → 통과 → 부채 해소 → 진행
```

### 시나리오 B — 파이프라인/자동화 사용자

```
CI 파이프라인 안에서 에이전트가 자동으로 코드 수정 실행.
EDC가 설정된 부채 임계값(예: HIGH 2건 이상) 초과 시 자동 block.
exit code 1 반환 → CI 실패 → PR 머지 불가.
로그에 "미해소 인지부채: [목록]" 출력.
```

### 시나리오 C — 팀 코드 리뷰

```
에이전트가 작업한 커밋에 EDC 리포트가 첨부됨.
리뷰어가 `edc report` 실행 → 해당 작업에서 에이전트가
근거 없이 내린 결정 목록 + 해소 여부 확인.
"이 부분은 에이전트가 추정으로 넘어갔고, 테스트 없이 병합됨" 가시화.
```

## 4. MVP 범위

### 핵심 구성요소

#### ① 인지부채 감지기 (Debt Detector)

- 에이전트 출력 텍스트에서 불확실성 패턴 매칭
- 규칙 기반 정규식: `I think`, `probably`, `should work`, `might be`, `assume`, `아마`, `것 같`, `~일 수 있` 등
- 행동 리스크 레벨 분류:
  - `LOW` — 조언/설명
  - `MEDIUM` — 코드 제안
  - `HIGH` — 파일 수정/명령 실행

#### ② 부채 레지스트리 (Debt Registry)

- 로컬 파일: `.edc/debt.json`
- 스키마:
  ```json
  {
    "id": "string",
    "claim": "string",
    "risk_level": "LOW | MEDIUM | HIGH",
    "resolved": false,
    "evidence": "string | null",
    "timestamp": "ISO8601"
  }
  ```

#### ③ 게이트 (Gate)

- Claude Code hook (`PreToolUse`)으로 `Edit`, `Write`, `Bash` 직전 인터셉트
- 미해소 HIGH 부채 존재 시 → 실행 거부 + 이유 출력
- `--force` 플래그로 의도적 override 허용 (막는 게 아니라 의식적 선택)

#### ④ CLI 인터페이스

```bash
edc list              # 현재 미해소 부채 목록
edc resolve <id>      # 부채 해소 (근거 텍스트 또는 test pass 증거 첨부)
edc report            # 세션 전체 부채 리포트
edc gate --check      # 현재 게이트 통과 가능 여부
```

### 기술 스택

- **Python 3** 단일 스크립트 (또는 Node.js)
- Claude Code **Hooks** — `settings.json`의 `PreToolUse`, `PostToolUse`
- 로컬 JSON 파일 저장소
- 정규식 기반 감지 (LLM 호출 없음 — 데모 안정성 우선)

---

## 5. 절대 하지 말아야 할 과한 범위

| 하고 싶어지는 것 | 왜 하지 말아야 하는가 |
|---|---|
| LLM으로 불확실성 의미론적 분석 | 레이턴시 + 비용, 데모 불안정 |
| 웹 대시보드 | 하루 안에 완성 불가, 핵심 가치와 무관 |
| 멀티 에이전트 실제 오케스트레이션 | 철학은 맞지만 구현은 주 단위 작업 |
| VS Code 익스텐션 | CLI로 충분히 컨셉 증명 가능 |
| 클라우드 동기화/팀 공유 | 로컬 JSON으로 먼저 증명 |
| 플러그인 시스템 | 추상화 과잉, 하나만 잘 |
| 자동 부채 해소 판단 | 에이전트가 스스로 해소 판단하면 컨셉 붕괴 |

## 데모 핵심 장면

Claude Code가 **"아마 이 함수가 문제일 것 같습니다"** 라고 말한 뒤 파일을 수정하려는 순간, EDC가 가로채서 **"인지부채 미해소 — 테스트 실행 후 진행하세요"** 를 출력하는 장면 하나면 충분하다.
